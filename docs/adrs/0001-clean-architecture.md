# ADR 0001: Clean Architecture

## Estado
Aceptado

## Contexto
Se requeria una solucion mantenible y testeable con separacion clara de responsabilidades.

## Decision
Se adopto Clean Architecture con 4 capas: Api, Application, Domain, Infrastructure.

## Consecuencias
Separacion clara y facil de mantener.  
Tests unitarios e integracion mas simples.  
Mayor estructura inicial (mas proyectos/clases).
